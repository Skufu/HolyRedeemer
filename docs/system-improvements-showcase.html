1. Circulation Status Tracking

Enhanced

Granular Transaction Lifecycle

Added circulation_status column to track the full lifecycle of every book transaction independently from the base transaction status.

  • New column: circulation_status (active, returned, overdue, lost, damaged)
  • Receipt number generation for every transaction (format: RCT-YYYYMMDD-XXXX)
  • Incident type tracking: none, damage, lost
  • Replacement cost applied field for financial tracking
  • Incident details text field for librarian notes
ALTER TABLE transactions ADD COLUMN circulation_status VARCHAR(20) DEFAULT 'active'; ALTER TABLE transactions ADD COLUMN receipt_no VARCHAR(50) UNIQUE; ALTER TABLE transactions ADD COLUMN incident_type VARCHAR(20) DEFAULT 'none';

Transaction Flow with Status

Checkout → Active → Returned / Overdue → Lost / Damaged → Resolved

3. Damage & Loss Reporting Module

New

Incident Tracking System

Complete module for reporting, tracking, and resolving damaged or lost book incidents with receipt numbers and financial accountability.

  • New damage_lost_incidents table with full audit trail
  • Incident types: damage, lost with severity levels (minor, moderate, severe, total_loss)
  • Unique receipt numbers for every incident (format: INC-YYYYMMDD-XXXX)
  • Automatic fine creation for lost/damaged books
  • Copy status updates: lost, damaged
  • Replacement cost calculation and tracking
  • Status workflow: pending → assessed → resolved / disputed

Incident Reporting Flow

Librarian Reports → Generate Receipt No → Assess Severity → Create Fine → Update Copy Status → Resolve Incident

UI Components

  • Admin Page: Full incident management with report, view, resolve capabilities
  • Librarian Page: Incident reporting and view-only incident list
  • Filterable table: by incident type, status, student search
  • Detail dialog with full incident information
  • Summary cards: Total incidents, pending, resolved, total cost
  • Color-coded severity and status badges

4. Reservation Auto-Cancel on Borrow

New

Automatic Reservation Management

When a book is checked out, all pending reservations for that book are automatically cancelled to prevent conflicts and maintain accurate availability.

  • New columns: auto_cancelled_at, reservation_queue_position
  • Auto-cancels all pending reservations when a book is borrowed
  • Queue position tracking for reservation ordering
  • Automatic queue position renumbering after cancellations
  • Next pending reservation lookup for notification when book is returned
-- Auto-cancel on checkout UPDATE book_requests SET status = 'cancelled', auto_cancelled_at = NOW() WHERE book_id = $1 AND status = 'pending' AND request_type = 'reservation';
