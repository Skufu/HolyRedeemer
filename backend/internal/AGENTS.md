# AGENTS.md - backend/internal

## OVERVIEW
High-complexity internal core managing business logic, data persistence, and domain security.

## STRUCTURE
- `handlers/`: Domain-based HTTP handlers (Gin) implementing library business logic.
- `repositories/sqlcdb/`: Type-safe repository layer generated by `sqlc`.
- `database/`: SQL migrations (`migrations/`) and `sqlc` query definitions (`queries/`).
- `middleware/`: Auth (JWT), Logging, and Role-Based Access Control (RBAC).
- `utils/`: Core utilities for JWT, Passwords, and custom HR-specific QR codes.
- `testutil/`: Standardized test setup, database mock helpers, and test fixtures.

## WHERE TO LOOK
- **Domain Logic:** Create/edit files in `handlers/` (e.g., `books.go`, `circulation.go`).
- **DB Queries:** Edit `.sql` files in `database/queries/`, then run `make sqlc`.
- **Role Permissions:** RBAC definitions in `middleware/auth.go` and usage in `handlers/`.
- **Tx Implementation:** Search for `db.Begin` in `circulation.go` or `books.go`.
- **QR Code Logic:** Format and generation rules in `utils/qr_code.go`.

## CONVENTIONS
- **Transaction Pattern:** `tx, _ := h.db.Begin(ctx)` -> `defer tx.Rollback(ctx)` -> `queries.WithTx(tx)`.
- **Row Locking:** ALWAYS use `...ForUpdate` queries for circulation to prevent race conditions.
- **PgType Safety:** Use `toPg*`/`fromPg*` helpers in `handlers/helpers.go` for all DB conversions.
- **Handler DI:** Handlers must receive `*pgxpool.Pool` to support transactional writes.
- **Date Format:** Use `2006-01-02` for backend dates; `time.RFC3339` for timestamps.

## ANTI-PATTERNS
- **Non-Transactional Writes:** Performing multiple DB writes without a transaction (e.g., `fines.go`).
- **Manual UUID Parsing:** Re-implementing UUID validation; use `uuid.Parse` and `toPgUUID`.
- **Role Omission:** Skipping `RequireRoles` middleware for sensitive librarian/admin actions.
- **Direct DB Access:** Bypassing `sqlcdb` queries for manual string-concatenated SQL.
- **Ignoring Rollbacks:** Failing to `defer tx.Rollback()` when using transactions.
