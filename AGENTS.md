**Generated:** 2026-01-21
**Tech:** Go (Gin) + React (Vite) + PostgreSQL (Neon)

## Quick Reference

**Frontend:**
```bash
cd frontend
npm run dev          # http://localhost:4127
npm run test:coverage # Vitest + MSW
npm run lint
```

**Backend:**
```bash
cd backend
make dev            # Air hot-reload on :8080
make test           # All Go tests
make sqlc           # Regenerate after SQL changes
make fmt; make lint # golangci-lint
make migrate-up
```

## Anti-Patterns (FORBIDDEN)

| Pattern | Why | Consequence |
|---------|------|------------|
| Edit `sqlcdb/*.go` files | Generated by sqlc, changes lost on regeneration | Data loss, build breakage |
| Ignore errors (`_ =`) | Golangci-lint enforces, AGENTS.md forbids | Lint failure |
| Multi-step writes without transactions | Data consistency risk (e.g., PayFine) | Partial updates, corrupt state |
| Commit `*.exe`, `*.log`, `*.original.tsx` | Gitignore exists but files committed | Dirty repository |
| Skip Vitest with `.skip` extension | Use `.skip` flag or `it.skip()` instead | Non-standard test skipping |
| Direct type assertions without `toPg*`/`fromPg*` helpers | Postgres pgtype conversion issues | Runtime panics, wrong data |

## Project-Specific Patterns

**Backend:**
- **Response format:** Always use `pkg/response` package (`response.Success()`, `response.BadRequest()`)
- **Pagination:** Dual queries (data slice + total count), meta includes `total_pages`
- **pgtype helpers:** Import `handlers/helpers.go` for `toPgUUID`, `fromPgText`, `fromPgNumeric`
- **Transaction pattern:** `db.Begin(ctx)` → `defer tx.Rollback(ctx)` → `queries.WithTx(tx)` → `tx.Commit(ctx)`
- **Handler constructor:** Must receive `*pgxpool.Pool` for transaction support
- **Row locking:** Circulation operations use `GetCopyByIDForUpdate` to prevent race conditions

**Frontend:**
- **Imports:** Use `@/` alias (`import { booksService } from '@/services/books'`)
- **Services:** Each domain has `src/services/{domain}.ts` with `list`, `get`, `create`, `update`, `delete`
- **Hooks:** TanStack Query in `src/hooks/`, mutation format: `useCreateX`, `useUpdateX`, `useDeleteX`
- **State:** Server state = TanStack Query, client state = Zustand (`authStore`, `userSettingsStore`)
- **API interceptor:** `services/api.ts` handles auto-refresh on 401, retries failed requests
- **Error handling:** Use `getErrorMessage()` from `services/api` for toast messages
- **Styling:** Tailwind + shadcn/ui, delegate visual changes to `frontend-ui-ux-engineer`

## Critical Gotchas

- **Port mismatch:** Frontend dev server is `4127` (not default 5173), database is `5433` (not 5432)
- **Missing transactions:** `fines.go` → `PayFine` lacks transaction despite CreatePayment + UpdateFineStatus writes
- **SQL changes:** ALWAYS run `make sqlc` after modifying `internal/database/queries/*.sql`
- **Token storage:** `localStorage.getItem('lms_access_token')`, `lms_refresh_token` (7 days)
- **Environment:** `VITE_API_URL=http://localhost:8080` (frontend), `PORT=8080` (backend)
- **Date format:** Backend `"2006-01-02"`, Frontend `date-fns` library
- **Roles:** `super_admin`, `admin`, `librarian`, `student` - enforced at middleware level
- **Hot reload on Windows:** `setup_and_run.sh` detects MINGW/CYGWIN, skips Air (uses `go run` instead)

## Database Schema Notes

- **Transactions:** Required for Checkout, Return, Renew, Create Book (creates copies atomically)
- **Row locking:** Used in circulation to prevent double-bookings
- **Generated models:** `internal/repositories/sqlcdb/models.go` - DO NOT EDIT
- **Migration tool:** `goose` in `internal/database/migrations/`
