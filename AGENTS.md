## STRUCTURE

```
HolyRedeemer/
├── backend/               # Go 1.24 + Gin REST API
│   ├── cmd/server/        # Entry point (main.go)
│   └── internal/          # Private packages
│       ├── handlers/      # HTTP handlers (fat - business logic here)
│       ├── database/      # Migrations + SQL queries for sqlc
│       ├── repositories/  # Generated sqlc code (DO NOT EDIT)

├── frontend/              # React 18 + TypeScript + Vite
│   └── src/

│       ├── hooks/         # TanStack Query wrappers

```

## COMMANDS

```bash
# Quick Start
./setup_and_run.sh --setup    # First time setup (DB + deps + migrations)
./setup_and_run.sh --run      # Start both backend + frontend

# Frontend (port 4127)
cd frontend
npm run dev                   # Dev server
npm run test                  # Vitest watch
npm run build                 # Production build

# Backend (port 8080)
cd backend
make dev                      # Hot reload with Air
make test                     # Run all tests
make sqlc                     # Regenerate after SQL changes
make migrate-up               # Apply migrations
make reset-db                 # Drop + remigrate
```

## CONVENTIONS

### Frontend
- **Imports**: Use `@/` alias → `import { x } from '@/services/books'`
- **Services**: Object pattern → `booksService.list()`, `booksService.create()`
- **Hooks**: `use{Domain}` → `useBooks()`, `useCirculation()`
- **State**: TanStack Query for server, Zustand for auth only
- **Styling**: Tailwind + shadcn/ui. Visual changes → delegate to `frontend-ui-ux-engineer`
- **Errors**: Use `getErrorMessage()` from `services/api`

### Backend
- **Handlers**: PascalCase exported, `{Domain}Handler` struct
- **Response**: Always use `pkg/response` → `response.Success()`, `response.BadRequest()`
- **DB Types**: Use helpers → `toPgUUID()`, `fromPgText()`, `toPgNumeric()`

### API Response Format
```json
{ "success": true, "data": {...}, "meta": { "page": 1, "total": 100 } }
{ "success": false, "error": { "code": "ERROR", "message": "..." } }
```

## CRITICAL PATTERNS

### Transactional Operations (MANDATORY for multi-step DB ops)
```go
tx, err := h.db.Begin(c.Request.Context())
if err != nil { return }
defer tx.Rollback(c.Request.Context())  // Auto-rollback

queries := h.queries.WithTx(tx)
// ... all operations using queries ...

if err := tx.Commit(c.Request.Context()); err != nil { return }
```

**Use for:** Checkout, Return, Renew, Create Book with copies

### Handler Struct Pattern
```go
type Handler struct {
    queries *sqlcdb.Queries
    db      *pgxpool.Pool  // Required for transactions
    config  *config.Config
}
```

## ANTI-PATTERNS

- **NEVER** edit `backend/internal/repositories/sqlcdb/*` - generated by sqlc
- **NEVER** use `as any`, `@ts-ignore`, `@ts-expect-error` in TypeScript
- **NEVER** ignore transaction errors with `_ =`
- **NEVER** commit `.env` files
- **NEVER** put business logic in SQL queries - keep in handlers
- **NEVER** use inline styles in React - use Tailwind classes
